# GA's conceptual process (overview)

# Population, individual and general terminology

  • At each search iteration we consider a set of solutions (population) instead of a single solution — different from Simulated Annealing and Hill Climbing.
  • Candidate solutions (individuals) must be "strings of characters" of prefixed length.
    Note: in practice, the data structure used to represent a string of characters is usually a list and not actually a str.

Terminology changes a bit to approximate biological inspiration:

search algorithms Genetic Algorithms
solution ⟶ individual
set of solutions ⟶ population
iterations ⟶ generations
objective function ⟶ fitness function
solution encoding ⟶ individual's genotype
'actual' solution ⟶ individual's phenotype

# Selection Process

The objective behind a Selection operation is to simulate the idea of "survival of the fittest". Starting from an initial random population, at each generation it selects 1 individual to integrate the new population. However, when deciding which individual to include it gives greater weight to those with higher fitness.

  • Choices must be probabilistic.
  • Individuals with better fitness have a higher probability of being selected.
  • No individual in the population can have a probability = 0 of being selected.
  • Process is repeated until the new population is completed.
    N independent executions, where N is the size of the population.

Examples of selection algorithms:

  1. Fitness Proportionate Selection (aka "Roulette Wheel")
  2. Ranking Selection [next class]
  3. Tournament Selection [next class]

# Crossover (one point crossover)

Input ⟶ two individuals (Parents)
Output ⟶ two individuals (Offspring)

  1. Selection of a random point between characters.
  2. Exchange substrings

# Mutation (one point mutation)

Input ⟶ one individual
Output ⟶ one individual

  1. Selection of a random character.
  2. Mutation of the character: replacement with another admissible one chosen randomly.

# Parameters of Genetic Algorithms

  1. Length of strings ⟶ typically part of the problem's definition
  2. Population size ⟶ as large as possible
  3. Selection algorithm ⟶ tip: start with Tournament selection [once learned]
  4. Crossover operator ⟶ dependent of the encoding
  5. Crossover rate ⟶ large; ≈ 1
  6. Mutation operator ⟶ dependent of the encoding
  7. Mutation rate ⟶ small; ≈ 0
  8. Maximum number of Generations ⟶ as large as possible

# Remarks

  1. If N (number of individuals in population) is odd, many things can be done — for example, one individual is selected, mutated and put in P'.
  2. We can add Elitism
    A set of individuals with good fitness is copied in P' with no Selection → Crossover → Mutation
    Typically, we apply Elitism only to the best in the population.

# Termination Condition

A "good enough" solution was found
OR
A prefixed number of generations was executed
