# Program's building blocks and how they relate to Hill Climbing's search

When we break down the three main entities into an actual program, the building blocks we should think about are:

Decision variables
Constraint variable
Build solution function
Objective function
Solution admissibility function
Algorithm's Operator(s) function(s) —> e.g. Neighborhood function in Local Search
Search function (the algorithm)

Since our algorithm is Hill Climbing we will interconnect these building blocks by approximating its step-by-step instructions as follows:

  • 1. Initialize a random solution.
    translating to code: Call the Build Solution function to generate a boolean list, where values are randomly selected. Then use the Solution Admissibility Function to check if the created solution "respects" the problem's constraint. To complete this first step, we should already check the solution's fitness — using the Objective function — and store it in a variable (for example, current_solution_fitness)

  • 2. Repeat until stopping condition
    translating to code: Create a loop where its boolean expression checks at each iteration if the stopping condition was reached. We'll see later on what could be the stopping condition and why.

    • 2.1. Generate a y belonging to N(x) (i.e. a neighbor belonging to the current solution's neighborhood)
      translating to code: At this point, we should actually generate all neighbor solutions in the neighborhood at once, making it easier to check which one has the best fitness. We'll implement a specific function for that called generate_neighborhood. Note: just as the Objective function is actually broken down into two (one to calculate fitness and another to check admissibility), the Neighborhood function is also broken down into smaller parts. We'll look into it later on.

    • 2.2. If the fitness of y is better or equal to the fitness of x, then x := y
      translating to code: Once we generated all neighbor solutions in the previous step, we check all of their "fitness". If at least one of them has a higher fitness (remember its a maximization problem) then the "best" neighbor becomes the current solution. Here the subtasks are: use the Objective function to check each of the neighbors' fitness and store them in a list. Then check which fitness is the highest and return the corresponding neighbor (i.e. the Boolean list that evaluates to the highest fitness value). To conclude, check if the best neighbor's fitness is greater or equal to the current solution's fitness. If yes, then the best neighbor becomes the current solution (e.g. current_solution = best_neighbor) and we repeat the process. If no (i.e. all neighbors have worst fitness), then we stop the loop.

  • 3. Return x (i.e. current solution)
    translating to code: After we leave the loop, return the current solution.

# Visually: Hill Climbing flowchart

So... In code, the Search function, which serves as our "direct translation" of the algorithm (i.e. where we combine all building blocks), would look something like this:

# ALGORITHM (Hill Climbing's Search Routine)
def search(values, weights, constraint, search_iterations_limit):
    
    # 1. Initialize random solution.
    solution_size = len(values)
    solution = initialize_solution(solution_size, weights, constraint)
    solution_fitness = objective_function(solution, values)

    # 2. Repeat until — current_solution has best fitness in neighborhood – or – iterations limit is reached.
    for i in range(search_iterations_limit):

        # 2.1. Generate current_solution's neighborhood.
        neighborhood = neighborhood_function(solution, values, weights, constraint)
        # Get best neighbor and its fitness from the neighborhood.
        best_neighbor, best_neighbor_fitness = get_best_neighbor(neighborhood, values)

        # 2.2. If the best neighbor has a better fitness than the solution,
        # then it becomes the current solution.
        if best_neighbor_fitness >= solution_fitness:
            solution = best_neighbor.copy()
            solution_fitness = best_neighbor_fitness
        
        # If there are no neighbors with a better fitness than the current solution,
        # then we stop (i.e. break) the search.
        else:
            break
    
    # Once we exit the loop — either by reaching the limit of iterations
    # or for not having a better neighbor — we return the solution.
    return solution

def initialize_solution(solution_size, weights, constraint):

Build Solution function
Inputs –> Solution size: <integer value>
Purpose –> Function responsible to initialize a solution — returns a list of randomly selected booleans. The number of elements in this list is defined by the Solution size argument.

def build_solution(solution_size):
    solution = []
    possible_values = [True, False]
    for i in range(solution_size):
        randomly_selected_value = choice(possible_values)  # Note that `choice()` is an utility function that must be imported into the script. => `from random import choice`
        solution.append(randomly_selected_value)
    return solution

———————————
Solution Admissibility function
Inputs –> Solution: <booleans list> | Weights (from our decision variables): <integers list> | Constraint (max_weight value): <integer value>.
Purpose –> Check if solution's combined weight is under the contraint — i.e. under the maximum weight.
Outputs –> Boolean value, which "answers" if True (i.e. is admissible) or False.

def is_admissible(solution, weights, constraint):
    weight_sum = 0
    solution_size = len(solution)
    for i in range(solution_size):
        if solution[i]:  
            weight_sum += weights[i]
    return weight_sum <= constraint

# Solutions

Remember, in programming there are always several possible solutions to the same problem.

# 1. objective_function

Since the goal here was to calculate the solution's total value, the idea was to iterate through it and sum the value of each object included in the knapsack (i.e. objects that had a corresponding True in the solution's boolean list).

def objective_function(solution, values):
    fitness = 0
    solution_size = len(solution)
    for i in range(solution_size):
        if solution[i]:
            fitness += values[i]
    return fitness

# 2. generate_neighbor

A neighbor in our knapsack problem, was a "copied" solution where we inverted only one value (e.g. [True, False, False, True, True] became [False, False, False, True, True]).
This method is called "Bit flipping" — from the idea that we're flipping a binary digit, since boolean values closely relate to binaries (1 / 0 ≈ True / False).
To invert a boolean value, we could use the Logical "Not" operator — in Python its literally written not.

def generate_neighbor(solution, i):
    neighbor = solution.copy()
    neighbor[i] = not neighbor[i]  # bit flip
    return neighbor

# 3. get_best_neighbor

With get_best_neighbor we wanted to loop through the neighbors included in the neighborhood, checking their fitness with the objective_function.

So, neighborhood was a "list of boolean lists". E.g. neighborhood:

neighborhood = [
    [False, True, True, True, True],
    [False, False, True, True, True],
    [False, True, False, True, True],
    [False, True, True, False, True],
    [False, True, True, True, False],
    [True, True, True, True, False],
    ...
]

Once we had all neighbors' fitness stored in a list, we could "reach" for the highest one (maximization problem), and use its location in the fitness list to match with the location of the best_neighbor in the neighborhood — since they were generated in the same order.

At this point, we had created the best_neighbor's boolean list and calculated the best_neighbor's fitness. Since we would still need this fitness value in the Hill Climbing algorithm to check if the current solution has a better fitness compared to it, we can take advantage of having already calculated the neighbor's fitness and return both information from the get_best_neighor function — i.e. the best_neighbor and the best_neighbor_fitness.

It would look like this:

def get_best_neighbor(neighborhood, values):
    fitness_list = []
    for neighbor in neighborhood:
        neighbor_fitness = objective_function(neighbor, values)
        fitness_list.append(neighbor_fitness)
    best_neighbor_fitness = max(fitness_list)
    best_neighbor_index = fitness_list.index(best_neighbor_fitness)
    best_neighbor = neighborhood[best_neighbor_index]
    return best_neighbor, best_neighbor_fitness
